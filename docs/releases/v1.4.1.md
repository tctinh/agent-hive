# Release v1.4.1

Compare: [`v1.4.0...v1.4.1`](https://github.com/tctinh/agent-hive/compare/v1.4.0...v1.4.1)

---

## Highlights

### Hardened Release Pipeline and Publish Credentialing

`v1.4.1` is a release-infrastructure patch. The `v1.4.0` tag was built cleanly but both publish jobs (npm and VS Code Marketplace) failed silently due to expired secrets (`NPM_KEY` and `VSCE_PAT`). This release adds preflight checks that catch credential failures before the upload is attempted, and ships the corrected package to npm.

**Why it matters:** Silent publish failures leave the registry behind the code. The preflight guards added here (`npm whoami`, `npm access list collaborators`, `npx @vscode/vsce verify-pat`) make credential problems a hard gate rather than a post-publish surprise.

### Fixed `opencode-hive` npm Package Contents

The `templates/` directory was missing from the published npm artifact in `v1.4.0` because it was not listed in `package.json`'s `files` whitelist. This is now corrected.

**Why it matters:** Consumers relying on `templates/mcp-servers.json` or `templates/context/tools.md` from the installed package would receive an incomplete installation silently.

### Version-Aware Release Artifact Test

`release-artifacts.test.mjs` now reads the version dynamically from the root `package.json` instead of using a hardcoded string, and asserts that `dist/index.js`, `skills/`, and `templates/` are all present in the npm pack dry-run output.

## What Changed

### Fixed: `opencode-hive` npm package now includes `templates/`
- Added `"templates/"` to the `files` array in `packages/opencode-hive/package.json`
- Consumers installing from npm now receive `templates/mcp-servers.json` and `templates/context/tools.md`

### Fixed / Hardened: release workflow credential preflights
- GitHub Actions `release.yml`: added `npm whoami` + `npm access list collaborators opencode-hive --json` before `npm publish`
- GitHub Actions `release.yml`: added `npx @vscode/vsce verify-pat tctinh` before VS Code Marketplace publish
- Either preflight failing now aborts the job with a clear error before any upload is attempted

### Hardened: release artifact test is now version-aware
- `release-artifacts.test.mjs`: reads version from `package.json` at runtime instead of hardcoding `1.3.6`
- Added npm pack dry-run assertions for `dist/index.js`, `skills/`, `templates/mcp-servers.json`, `templates/context/tools.md`
- `package.json` `release:check` script now runs the test as part of the pre-tag safety net

### Docs: `docs/RELEASING.md` rewritten
- Now version-agnostic (uses `vX.Y.Z` placeholders throughout)
- Added missed-release recovery section documenting the `v1.4.0` failure pattern and recommended recovery path
- Added credential preflight checklist as a required step before any tag push
