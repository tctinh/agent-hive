# Release v1.4.0

Compare: [`v1.3.1...v1.4.0`](https://github.com/tctinh/agent-hive/compare/v1.3.1...v1.4.0)

---

## Highlights

### Human-Facing Overview as Reserved Context

Agent Hive now treats `.hive/features/<feature>/context/overview.md` as a reserved, human-facing summary document. Agents maintain it through the existing `hive_context_write({ name: 'overview', content })` tool path, so the system gains a readable overview without introducing a parallel overview-write API.

**Why it matters:** Humans can review a concise overview-first artifact without losing the stronger execution contract in `plan.md`. The overview stays visible across status and VS Code surfaces, while worker execution still depends on `plan.md` and generated task specs.

### Overview-First Review and Approval Flow

Reviews are now document-aware: `plan` and `overview` comments are tracked separately, surfaced separately, and both can block approval until unresolved threads are cleared. The VS Code extension and OpenCode plugin both report those document-specific counts and guide users through the overview-first review flow.

**Why it matters:** The system now matches the intended human workflow: review the overview first, use the plan for deeper technical detail, and keep approval gates honest when either surface still has open feedback.

## What Changed

### New: Reserved `context/overview.md` workflow
- `hive-core` context service reserves `overview` as a special context file
- Worker execution context excludes reserved overview content while preserving all other context behavior
- Planner/orchestrator guidance now tells agents to refresh overview after significant plan changes and milestones

### New: Document-aware review storage
- Added review service support for `plan` vs `overview` review threads
- Approval checks now aggregate unresolved comments across both documents
- Feature/status metadata now exposes overview presence and per-document review counts

### Updated: Human-facing product surfaces
- `hive_status` reports overview metadata and overview-aware next actions
- VS Code sidebar shows Overview separately and avoids duplicate listing under generic Context
- Opening a feature now prefers overview when present and falls back to `plan.md` for legacy features

### Updated: Verification and release readiness
- Stabilized the `writeAtomic()` regression test to avoid environment-dependent readonly-directory failures
- Added targeted regressions for prompt guidance, plan approval, status reporting, and plugin behavior
- Bumped root and package versions to `1.4.0`

---

## Breaking Changes

None. `plan.md` remains the execution contract, existing context tooling is reused, and legacy features without `context/overview.md` still fall back to the prior plan-first behavior.

---

## Upgrade Notes

- No migration is required for existing features.
- To add a human-facing summary for an existing feature, write `context/overview.md` via `hive_context_write({ name: 'overview', content })`.
- If your review workflow assumed all comments lived in a single plan-only store, update any external tooling to read document-aware review counts.
