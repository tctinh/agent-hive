## Highlights

### PR #62 History Alignment Lands on the 1.3.x Patch Line

`v1.3.4` is the corrective patch release that formally documents PR #62 / `488aa29b76a2f10d74e8c9ee640541a53527fe66` on top of the already-shipped `1.3.x` release line. The release note explains the reserved `context/overview.md` model, keeps `plan.md` as the execution contract, and records why this patch exists after `v1.3.3`.

**Why it matters:** The release branch history never literally contained the PR #62 squash commit, so earlier notes could not honestly say that PR #62 shipped in `v1.3.2`. `v1.3.4` is where that history alignment is described explicitly and transparently.

### History Alignment, Not New Conflict Resolution

PR #62 commit `488aa29` and the shipped `1.3.x` release branch tip are tree-equivalent: they resolve to the same source tree even though their git histories differ. That means `v1.3.4` is not about new code conflict resolution; it is about release-branch history alignment, changelog correction, and technically honest release documentation.

**Why it matters:** Users get an accurate story about what changed on the patch line without being misled into thinking a hidden source merge or unreleased feature payload appeared after `v1.3.3`.
