# Release v1.3.2

Compare: [`v1.3.1...v1.3.2`](https://github.com/tctinh/agent-hive/compare/v1.3.1...v1.3.2)

---

## Highlights

### GitHub Copilot Rewrite Backported to the Release Line (PR #57)

This patch release pulls the large VS Code / GitHub Copilot rewrite from PR #57 onto the `release/human-overview-context-v1` branch for the `1.3.x` line. That brings the updated Copilot-facing generators, language-model tool wiring, sidebar/watch behavior, and extension packaging changes into the same branch already carrying the human-overview workflow work.

**Why it matters:** Teams using the release branch get the newer Copilot integration now, without giving up the branch-specific UX and workflow behavior they were already validating for the `1.3.x` line.

### Human-Facing Overview Workflow Preserved (PR #62)

The branch keeps the reserved `.hive/features/<feature>/context/overview.md` workflow introduced through PR #62. Overview remains the primary human-facing summary/history surface, while `plan.md` stays the execution contract for workers and generated task specs.

**Why it matters:** Humans can review a concise overview-first artifact while agents still execute against the stricter planning contract. The release keeps that separation intact instead of regressing to a plan-only UI.

### Release-Branch Reconciliation Includes #63 / `ac7e78d`

Because PR #57 branched before commit `ac7e78d` (`Feat: Custom commit and merge messages (#63)`), this release explicitly preserves that behavior while reconciling the two lines. Optional custom `message` support for `hive_worktree_commit` and `hive_merge` remains available across `hive-core`, `opencode-hive`, and `vscode-hive`.

**Why it matters:** The backport is not just a file merge. It keeps the release branch's already-integrated workflow behavior and the newer custom commit/merge message support together in one coherent patch release.

## Breaking Changes

None. This is a backported patch release on top of `v1.3.1`. The release keeps the human-overview workflow, preserves custom commit/merge message support from #63, and does not introduce a new migration requirement for existing `.hive/` feature directories.

---

## Upgrade Notes

- Upgrade targets the `1.3.x` line directly: use `v1.3.2` as the release tag and body for this branch.
- If you rely on human-facing feature summaries, continue writing `context/overview.md` via `hive_context_write({ name: 'overview', content })`.
- If you consume Hive review metadata externally, ensure your tooling understands document-aware review counts for both `plan` and `overview`.
- If you use custom commit or merge messages in automation, the `#63` / `ac7e78d` support remains present in this release branch.
