## Highlights

### Custom Multi-Model Subagents (PR #53)

Agent Hive now supports user-defined custom subagents. You can configure additional workers and reviewers in `~/.config/opencode/agent_hive.json` under the `customAgents` key. Each custom agent is derived from either `forager-worker` or `hygienic-reviewer`, inherits the base prompt and tool permissions, and lets you override `model`, `temperature`, `variant`, and `autoLoadSkills`.

**Why it matters:** Different models excel at different tasks. Codex-based models can be better for algorithmic backend work. Gemini can be better for UI/UX tasks. Claude Sonnet can be better for test authoring. Now you can route tasks to the right model automatically — orchestrators see your configured agents and their descriptions and choose the best match.

**Example config:**

```json
{
  "customAgents": {
    "forager-ui": {
      "baseAgent": "forager-worker",
      "description": "Use for UI-heavy or visual design focused implementation tasks.",
      "model": "anthropic/claude-sonnet-4-20250514",
      "temperature": 0.2,
      "variant": "high"
    },
    "reviewer-security": {
      "baseAgent": "hygienic-reviewer",
      "description": "Use for security-focused review passes."
    }
  }
}
```

**Inheritance:** Omitted fields resolve from the effective base-agent config, including user overrides in `agents`. `autoLoadSkills` merges with base skills and deduplicates.

**Guardrails:** Reserved names (built-in Hive agents, plugin aliases, OpenCode operational agent IDs) cannot be used as custom agent IDs. Custom agent descriptions must be non-empty.

### Blocked-Resume Retry Loop Prevention (PR #54)

Orchestrators running in scenarios with state drift (e.g., after compaction) would sometimes retry `hive_worktree_create({ ..., continueFrom: "blocked" })` on tasks that were not actually in `blocked` state, causing infinite dead-end loops.

This release closes that loop by splitting the worktree API into two distinct tools:

- **`hive_worktree_start`** — Normal starts for `pending` or `failed` tasks. This is the tool to call when starting work on a task.
- **`hive_worktree_create`** — Exclusively for resuming `blocked` tasks (`continueFrom: "blocked"` + `decision`). Calling it on a non-blocked task now returns a hard `terminal: true` error with `reason: "task_not_blocked"` and `correctTool: "hive_worktree_start"` so orchestrators self-correct immediately.

All orchestrator prompts, skills, docs, and e2e tests have been updated to reflect the split.

## Breaking Changes

None. `hive_worktree_create` is still available and still works for blocked resumes. The only observable behavior change is that calling `hive_worktree_create` on a non-blocked task now returns a terminal error instead of silently attempting to start the task — which was already incorrect behavior that led to loops.

---

## Upgrade Notes

- No migration needed for existing `.hive/` feature directories.
- If you have custom agent scripts that call `hive_worktree_create` for normal (non-blocked) starts, update them to call `hive_worktree_start` instead.
- To use `customAgents`, add the `customAgents` key to your `~/.config/opencode/agent_hive.json`. See the config example above or the updated README for details.
